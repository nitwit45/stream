Setting up Google SSO (Single Sign-On) in Next.js is most efficiently done using Auth.js (formerly known as NextAuth.js). It handles the complex OAuth 2.0 flow, session management, and security for you.

Here is the step-by-step guide to getting it running.

1. Create Google OAuth Credentials
Before writing code, you need to register your app with Google to get a Client ID and Client Secret.

Go to the Google Cloud Console.

Create a Project (or select an existing one).

Navigate to APIs & Services > OAuth consent screen.

Choose External (unless you only want people in your workspace to log in).

Fill in the required app name and support email.

Navigate to APIs & Services > Credentials.

Click Create Credentials > OAuth client ID.

Select Web application as the type.

Authorized JavaScript origins: Add http://localhost:3000.

Authorized redirect URIs: Add http://localhost:3000/api/auth/callback/google.

Copy your Client ID and Client Secret.

2. Install Auth.js
In your Next.js project folder, run:

Bash
npm install next-auth@beta
(Note: @beta is currently recommended for Next.js 14/15 App Router support).

3. Configure Environment Variables
Create a .env.local file in your root directory. Never commit this file to GitHub.

Code snippet
AUTH_GOOGLE_ID=your_google_client_id
AUTH_GOOGLE_SECRET=your_google_client_secret
AUTH_SECRET=your_random_secret_string
Tip: You can generate a secure AUTH_SECRET by running npx auth secret in your terminal.

4. Set Up the Auth Configuration
Create a file at auth.ts (usually in your root or src folder) to initialize the Google provider.

TypeScript
import NextAuth from "next-auth"
import Google from "next-auth/providers/google"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [Google],
})
5. Create the API Route
To handle the sign-in and callback requests, create a "catch-all" route. Create the following file:

app/api/auth/[...nextauth]/route.ts

TypeScript
import { handlers } from "@/auth" // Link to your auth.ts file
export const { GET, POST } = handlers
6. Add Sign-In to Your UI
You can now use the signIn function in a Server or Client Component.

Example: Sign-In Button (Server Component)
TypeScript
import { signIn } from "@/auth"

export default function SignInPage() {
  return (
    <form
      action={async () => {
        "use server"
        await signIn("google")
      }}
    >
      <button type="submit">Signin with Google</button>
    </form>
  )
}
Example: Protecting a Page
To check if a user is logged in, use the auth() function:

TypeScript
import { auth } from "@/auth"

export default async function Dashboard() {
  const session = await auth()

  if (!session) return <div>Please log in to view this page.</div>

  return (
    <div>
      <h1>Welcome, {session.user?.name}</h1>
      <img src={session.user?.image} alt="Profile" />
    </div>
  )
}
Deployment Note
When you deploy (e.g., to Vercel), remember to:

Update your Authorized Redirect URIs in Google Cloud Console to include your production URL (e.g., https://your-site.com/api/auth/callback/google).

Add your production URL as an environment variable: AUTH_URL=https://your-site.com.

Gemini said
For most developers and hobby projects, the answer is no. You can set this up entirely for free.

Here is the breakdown of where costs could appear as you scale, but why they won't hit you right now:

1. Google Cloud (The SSO Provider)
Cost: $0

Using Google as an OAuth provider for "Sign in with Google" is free. Unlike some enterprise "SSO" features (like SAML), standard OAuth 2.0 for user authentication does not have a monthly fee or a per-user charge for standard web apps.

Note: If you were using Google Cloud Identity Platform (a more complex service for managing millions of users), it is free for the first 50,000 monthly active users.

2. Auth.js / NextAuth.js (The Library)
Cost: $0

This is an open-source library. There are no licensing fees, no matter how many users you have.

3. Vercel (The Hosting)
Cost: $0 (Hobby Plan)

If you deploy your Next.js site to Vercel, the Hobby Plan is free for personal projects.

When you'd pay: If you move to a Pro Plan ($20/month) for commercial use or if your site exceeds 100GB of bandwidth.

4. Database (Optional but Recommended)
Cost: $0

If you want to save user data (like their profile picture or preferences), you'll need a database.

Free Options:

Supabase: Very generous free tier (500MB database).

Neon: Great serverless Postgres with a free tier.

MongoDB Atlas: Free shared clusters.

Summary Table
Component	Free Tier	When it costs money
Google SSO	Unlimited users	Never (for standard OAuth)
Hosting (Vercel)	100GB Bandwidth	$20/mo (Commercial/Pro)
Database (Supabase)	50,000 Users	Over 50k Users / 500MB storage
SSL Certificate	Included Free	Never
Verdict: For your personal projects or your final year IoT project, you won't need to spend a cent to get Google SSO working.
